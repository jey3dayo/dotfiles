統合開発支援コマンドの包括的なドキュメントとガイドです。

**総コマンド数**: 38個 + 7個の共通ユーティリティ
**総仕様行数**: 8,635行
**対応言語**: TypeScript, Go, Python, Rust など
**実装状態**: ⚠️ 仕様定義のみ（実装は未完了）

1. **`/task`** - インテリジェント・タスク・ルーター

4. **`/learnings`** - AI駆動知見記録
   - カテゴリ別学習記録
   - 自動提案システム
   - 知見の検索・共有

## 🧭 コマンドメタデータ

すべてのClaudeコマンドはファイル先頭のYAMLフロントマターでメタ情報を宣言します。

- `description` (必須): コマンドの目的を1行で記述
- `argument-hint` (必須): 主要な引数やフラグを簡潔に記載
- `allowed-tools` (任意): 利用を想定するツールを明示
- メタデータと本文の仕様が乖離しないように更新時は両方を揃える

## 🏗️ アーキテクチャ設計

1. **統一コンテキスト** - TaskContextによる一貫性
2. **共通ユーティリティ** - コード重複の排除
3. **インテリジェント選択** - タスク意図に基づく最適化

### データフロー

```
User Input
    ↓
TaskContext生成 (task-context.md)
    ↓
プロジェクト判定 (project-detector.md)
    ↓
エージェント選択 (agent-selector.md)
    ↓
Context7統合 (context7-integration.md) [オプション]
    ↓
タスク実行
    ↓
エラーハンドリング (error-handler.md)
    ↓
結果返却・学習記録
```
