# CLAUDE.md Design Principles

## 目的

グローバルなCLAUDE.mdを汎用的かつ保守しやすく保つための設計原則

## 設計原則

### 1. 責任の分離

- **CLAUDE.md**: 汎用的なガイドラインのみ
- **スキル**: 機能固有の詳細（コマンド、ワークフロー等）
- **steering**: プロジェクト固有のルール

### 2. 削除すべき情報

- スキル固有の詳細（コマンド一覧、ワークフロー等）
  - 理由: スキル自体が詳細を持っているため重複
- 動的に変わる情報（Installed Skills等）
  - 理由: 最新性を保つのが困難、使用時にスキルを参照すべき
- 特定のツール/フレームワークの詳細説明
  - 理由: スキルまたはsteeringに委譲

### 3. 維持すべき情報

- steeringの読み込み指示
  - 理由: プロジェクトコンテキストの取得に不可欠
- 汎用的な開発ルール
  - 理由: すべてのプロジェクトに適用可能
- 言語設定
  - 理由: ユーザー体験の基本設定

### 4. 汎用的なルールの特徴

- プロジェクト/ツールに依存しない
- 開発の基本姿勢を示す
- 自律性と効率性を促進

## 実装例

```markdown
# Development Guidelines

## Language

- 日本語で回答

## Project Context

プロジェクトに `.kiro/steering/` がある場合は読み込んで参照すること。

## Development Rules

- ユーザーの指示に従い、その範囲内で自律的に行動する
- 必要なコンテキストを収集し、作業を完遂する
- 質問は、本質的な情報が不足している場合や指示が致命的に曖昧な場合のみ
- 複数の独立したタスクがある場合は、サブエージェント（Task tool）による並列実行を検討
```

## 効果

- **保守性**: 更新が必要な箇所が明確
- **明快さ**: 役割分担が明確
- **拡張性**: 新しいスキルの追加が容易
- **一貫性**: 汎用ルールとプロジェクト固有ルールの分離
