# Agent Quality Checklist

## 構造

- [ ] YAMLフロントマター完全（name, description, tools, color, model）
- [ ] 役割が明確に定義
- [ ] 能力が明示的にリスト化
- [ ] 起動コンテキストを文書化
- [ ] 出力形式を仕様化

## 設計

- [ ] 単一の集中的な責任
- [ ] 役割に適したツールアクセス
- [ ] モデル選択が正当化（Sonnet vs Opus）
- [ ] カラーがエージェントタイプ/深刻度を反映
- [ ] 既存パターンに従う

## 統合

- [ ] 親コマンドを文書化
- [ ] 関連エージェントを識別
- [ ] 起動例を提供
- [ ] 出力形式が解析可能

## 出力品質

- [ ] 一貫した見出しで構造化
- [ ] 深刻度/優先度インジケーターを含む
- [ ] file:line参照を提供
- [ ] 推奨事項がアクション可能
- [ ] 要約セクションを含む

## テスト

- [ ] Task tool起動経由でテスト
- [ ] ツールアクセスを検証
- [ ] 出力形式を検証
- [ ] 親ワークフローとの統合を確認

## 準拠

- [ ] kebab-case命名規則
- [ ] 適切なディレクトリ/サブディレクトリに配置
- [ ] 既存エージェント機能の重複なし
- [ ] read-only原則を尊重
