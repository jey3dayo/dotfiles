# Rules Quality Checklist

## Rules

- [ ] 明確で曖昧さのない制約記述
- [ ] ルールが存在する理由の説明
- [ ] スコープを明確に定義（paths で適用範囲を限定）
- [ ] 正しい例を提供
- [ ] 不正な例を提供
- [ ] 例外を文書化（存在する場合）
- [ ] 強制メカニズムを指定

## 全般的な品質

- [ ] 既存ルールと競合しない
- [ ] プロジェクト哲学と一致
- [ ] 測定可能/検証可能
- [ ] 例が実際のプロジェクトパターンを使用

## 一貫性

- [ ] 命名規則に従う（kebab-case）
- [ ] 正しいディレクトリに配置
- [ ] テンプレート構造を使用
- [ ] 他のルールと一貫した用語
- [ ] 関連ルールを相互参照

## テスト可能性

- [ ] ルール違反が検出可能
- [ ] 強制メカニズムが動作
- [ ] フィードバックが明確でアクション可能
- [ ] エッジケースを考慮
