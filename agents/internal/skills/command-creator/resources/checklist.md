# Command Quality Checklist

## 構造

- [ ] YAMLフロントマター（description + argument-hint）
- [ ] 目的セクションで「なぜ」を説明
- [ ] 明確な使用例
- [ ] すべての引数・フラグを文書化
- [ ] エラー処理セクション

## 設計

- [ ] 単一の明確な責任
- [ ] 類似コマンドの既存パターンに従う
- [ ] Shared librariesを適切に活用
- [ ] 関連agent/skillと統合
- [ ] 長時間実行の場合はセッション管理

## ドキュメント

- [ ] 実世界の例を提供
- [ ] エッジケースを文書化
- [ ] 統合ポイントをリスト化
- [ ] トラブルシューティングセクションを含む

## テスト

- [ ] ローカルプロジェクトで最初にテスト
- [ ] 例が動作することを確認
- [ ] エラー条件をテスト
- [ ] Shared library統合を検証

## 準拠

- [ ] kebab-case命名規則
- [ ] 適切なサブディレクトリに配置
- [ ] 既存機能の重複なし
- [ ] read-only原則を維持
