# Distribution Bundle Structure Template

## Directory Layout

```
bundles/<bundle-name>/
├── skills/                          # Skill symlinks
│   ├── skill-1 -> ../../../skills-internal/skill-1
│   ├── skill-2 -> ../../../skills-internal/skill-2
│   └── external-skill -> ../../../skills/external-skill
├── commands/                        # Command symlinks
│   ├── command-1 -> ../../../commands-internal/command-1
│   └── command-group -> ../../../commands-internal/command-group/
│       ├── subcommand-1/
│       │   └── command.ts
│       └── subcommand-2/
│           └── command.ts
├── config/                          # Optional configuration
│   ├── shared-rules.md              # Shared Claude Code rules
│   ├── bundle-config.json           # Bundle metadata
│   └── templates/                   # Reusable templates
│       └── template.md
└── README.md                        # Bundle documentation
```

---

## Symlink Patterns

### Skills

```bash
# From: bundles/<bundle-name>/skills/
# To:   skills-internal/<skill-name>/
cd bundles/<bundle-name>/skills
ln -s ../../../skills-internal/<skill-name> ./
```

### Commands

```bash
# Single command
cd bundles/<bundle-name>/commands
ln -s ../../../commands-internal/<command-name> ./

# Command group (with subcommands)
ln -s ../../../commands-internal/<command-group> ./
```

---

## File Requirements

### Required Files

- **README.md**: Bundle purpose, contents, installation
- **skills/**: At least one skill symlink
- **commands/**: At least one command symlink

### Optional Files

- **config/shared-rules.md**: Claude Code instructions for bundle users
- **config/bundle-config.json**: Metadata (version, maintainers, etc.)
- **config/templates/**: Reusable templates for skills/commands

---

## Naming Conventions

- **Bundle name**: lowercase-with-hyphens (e.g., `frontend-tools`, `backend-utils`)
- **README.md**: Required at bundle root
- **config/**: Optional, use for shared configurations only

## Validation Commands

```bash
# Check structure
tree -L 2 bundles/<bundle-name>/

# Validate symlinks
find bundles/<bundle-name>/ -type l -exec test ! -e {} \; -print

# Count skills
ls -1 bundles/<bundle-name>/skills/ | wc -l

# Count commands
find bundles/<bundle-name>/commands/ -name "command.ts" | wc -l
```
