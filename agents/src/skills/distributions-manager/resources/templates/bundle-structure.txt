# Distribution Bundle Structure Template

## Directory Layout

```text
bundles/<bundle-name>/
├── skills/
│   ├── react -> ../../../src/skills/react
│   ├── polish -> ../../../src/skills/polish
│   └── my-custom-skill/
│       └── SKILL.md
├── rules/
│   ├── claude-md-design.md -> ../../../src/rules/claude-md-design.md
│   └── tools/
│       └── my-team-rule.md
├── agents/
│   ├── code-reviewer.md -> ../../../src/agents/code-reviewer.md
│   └── kiro -> ../../../src/agents/kiro
├── config/
│   ├── shared-rules.md
│   └── bundle-config.json
└── README.md
```

## Notes

- `skills/`, `rules/`, and `agents/` are the active bundled asset types
- `commands/` is intentionally omitted because bundled commands are not deployed by the current Home Manager module
- You can mix reused symlinks and bundle-specific directories/files

## Validation Commands

```bash
tree -L 2 bundles/<bundle-name>/
find bundles/<bundle-name>/ -type l -exec test ! -e {} \; -print
find bundles/<bundle-name>/skills -type l -exec sh -c '
  target=$(readlink -f "$1")
  test -f "$target/SKILL.md" || echo "Missing SKILL.md: $1"
' _ {} \;
```
